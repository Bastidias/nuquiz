# Kerberos Components

**Domain:** 5 — Identity and Access Management &nbsp;|&nbsp; **Pattern:** Dimensions &nbsp;|&nbsp; **Tags:** 5.6
**Status:** draft (SME review pending)

The seven Kerberos components CISSP candidates should discriminate by category (service, artifact, boundary, identity) and role. Paired Concept to `kerberos-flow` — this is the "what are the nouns" Concept; the flow is the "what are the verbs" Concept. CISSP exams regularly test the distinction between the KDC (the umbrella service) and its two sub-services (AS and TGS), and between a TGT (generic authentication proof) and a service ticket (per-target credential).

| Component | category | role | typical relationship |
|---|---|---|---|
| KDC | Service | Central authentication authority | Runs AS<br>Runs TGS |
| AS | Service | Issues TGTs [s1] | Sub-service of KDC |
| TGS | Service | Issues service tickets [s1] | Sub-service of KDC |
| TGT | Artifact | Proves initial authentication | Issued by AS<br>Presented to TGS |
| Service Ticket | Artifact | Proves service authorization | Issued by TGS<br>Presented to service |
| Realm | Boundary | Administrative domain [s1] | Contains principals |
| Principal | Identity | Named entity with a key [s1] | Authenticates to AS |

## Notes

- **Cell convention:** each `<br>`-separated item is one atomic Fact. **No parentheticals in cells.** Acronym expansions (KDC = Key Distribution Center, AS = Authentication Service, TGS = Ticket-Granting Service, TGT = Ticket-Granting Ticket) go in this section, not in cell values.
- **Acronym expansions.** `KDC` = Key Distribution Center. `AS` = Authentication Service. `TGS` = Ticket-Granting Service. `TGT` = Ticket-Granting Ticket. These are RFC 4120 [s1] canonical identifiers and are widely tested by name.
- **KDC is a role, not a process.** In practice the KDC is implemented as a single server process (e.g., `krb5kdc` in MIT Kerberos, the KDC role in Active Directory domain controllers) that answers both AS_REQ and TGS_REQ messages. The AS and TGS are *logical* sub-services of the KDC, not separate daemons. The separation matters because it partitions which long-term keys each interaction touches — AS uses the client's long-term key; TGS uses the TGS session key established in AS_REP.
- **TGT vs Service Ticket.** The TGT is a generic "I authenticated once to the AS" credential, valid for any service in the realm. The Service Ticket is a per-target credential usable only against the specific service it was issued for. The TGS issues Service Tickets; the AS does not. A question that asks "which component issues a ticket to a specific service" points to TGS, not AS.
- **Principal naming convention.** A Kerberos principal name is formatted `primary/instance@REALM` (e.g., `alice@EXAMPLE.COM` for a user, `HTTP/web.example.com@EXAMPLE.COM` for a service). The realm portion is always the last segment after `@` and is conventionally written in uppercase. CISSP rarely tests the `primary/instance` split but does test the `@REALM` suffix as the realm-qualification mechanism.
- **Realms and cross-realm trust.** A Kerberos realm is bounded by a single KDC (or KDC cluster sharing state). Cross-realm authentication requires a shared inter-realm key between the two realms' KDCs, producing a cross-realm TGT. Multi-realm topologies (transitive trust, hierarchical realms) are out of scope for this Concept.
- **Long-term keys vs session keys.** Principals hold *long-term keys* derived from their password (users) or randomly provisioned (services). Session keys are short-lived symmetric keys generated by the KDC for a specific exchange (TGS session key for AS↔TGS interactions, service session key for client↔service interactions). This distinction is what keeps the long-term key off the wire.
- **Out of scope for this Concept:** the Kerberos V5 message flow (separate Ordered Concept — `kerberos-flow`), PKINIT and certificate-based initial authentication, FAST (Flexible Authentication Secure Tunneling), constrained delegation and S4U2Self / S4U2Proxy, Windows-specific Kerberos extensions (PAC in MS-KILE), Kerberoasting and Golden Ticket attacks.
